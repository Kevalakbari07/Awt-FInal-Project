## 📌 PROJECT OVERVIEW

**Dairy Management System** is a comprehensive full-stack web application designed to help dairy farmers and administrators manage:
- Farmer registration and management
- Daily milk collection tracking
- Payment processing and approval
- System reports generation
- Government dairy schemes information

**Technology Stack:**
- **Frontend:** React 19 + TypeScript + Vite
- **Backend:** Node.js + Express.js + MongoDB/Mongoose
- **Database:** MongoDB (Local Instance)
- **Build Tool:** Vite (Fast refresh development)

**Deployment Status:** Ready for production deployment

---

## 🎯 PROJECT PURPOSE & GOALS

### Problem Statement
Dairy farmers need a centralized system to:
- Track daily milk collections
- Monitor payment status from dairy cooperatives
- Access government scheme information
- Stay informed about payment approvals

### Solution
A dual-role web application with:
1. **Farmer Interface** - Dashboard, milk tracking, payment viewing
2. **Admin Panel** - Farmer management, payment approval, report generation

### **API Routes** (17 Total Endpoints)

#### **Authentication Routes** (`/routes/auth.js` - 2 endpoints)

**1. POST /api/signup**
- Purpose: Register new user
- Request Body:
  ```json
  {
    "username": "farmer1",
    "password": "Farmer@123"
  }
  ```
- Response (201 Created):
  ```json
  {
    "message": "User created successfully",
    "username": "farmer1",
    "role": "user"
  }
  ```
- Validation:
  - Username must be unique (case-insensitive)
  - Password must be provided
  - Default role: "user" (unless username is astro@133790)
- Error Cases:
  - Username already exists → 400 Bad Request
  - Missing username/password → 400 Bad Request

**2. POST /api/login**
- Purpose: Authenticate user
- Request Body:
  ```json
  {
    "username": "farmer1",
    "password": "Farmer@123"
  }
  ```
- Response (200 OK):
  ```json
  {

## 🔄 DATA FLOW & USER JOURNEYS

### **Journey 1: User Signup & Login**
```
1. User visits frontend (http://localhost:5173)
2. Frontend shows Login page
3. User clicks "Sign Up"
4. Fills form: username, password, confirm password
5. Clicks "Register"
6. Frontend sends: POST /api/signup {username, password}
7. Backend validates:
   - Username unique? Check MongoDB
   - Password provided? Check
8. Backend creates User document in MongoDB
9. Returns: {message: "User created successfully", role: "user"}
10. Frontend redirects to Login
11. User enters credentials
12. Frontend sends: POST /api/login {username, password}
13. Backend verifies:
    - Username exists? Check MongoDB
    - Password matches? Check
14. Backend returns: {role: "admin" or "user"}
15. Frontend stores: localStorage.setItem("role", role)
16. Frontend routes to Dashboard
```

```
1. User logged in, visits /milk page
2. Page shows form:
   - Select Farmer (GET /api/farmers)

5. Frontend sends: POST /api/milk {farmer, quantity, fat, rate, total, date}
6. Backend validates all fields
7. Backend creates MilkCollection document in MongoDB
8. Returns success response
9. Frontend displays success message
10. Frontend refreshes milk list: GET /api/milk
11. Page shows updated collection history
```

7. Backend updates Payment document in MongoDB
8. Backend returns: {message: "Payment status updated"}
9. Frontend refreshes payment list: GET /api/payments
10. Page shows payment status as "Approved"
11. Farmer can now see approved payment in /payments page
```

### **Journey 4: View Dashboard Analytics**

1. User visits /dashboard

## 🔐 AUTHENTICATION & SECURITY SYSTEM

### **Authentication Flow**

**User Registration (Signup):**
```
Frontend Form
  ↓
User enters: username, password
  ↓
Frontend validation:
  - Username ≥ 4 characters
  - Password ≥ 6 characters, 1 uppercase, 1 number
  ↓
POST /api/signup {username, password}
  ↓
Backend checks:
  - Is username unique? (query MongoDB)
  - Are required fields present?
  ↓
If valid:
  - Save to User collection
  - Set role = "user" (or "admin" if username = "astro@133790")
  - Return 201 Created
  ↓
If invalid:
  - Return 400 Bad Request with error message
  ↓
Frontend redirects to Login
```

**User Login:**
```
Frontend Form
  ↓
User enters: username, password
  ↓
Frontend validation:
  - Fields not empty
  ↓
POST /api/login {username, password}
  ↓
Backend queries:
  - Find user by username in MongoDB
  ↓
If user found:
  - Compare provided password with stored password
  - If match: Return 200 OK with role
  - If not match: Return 400 "Invalid credentials"
  ↓
If user not found:
  - Return 400 "User not found. Please signup first."
  ↓
If MongoDB down:
  - Return 500 "Server is down. Please try again later."
  ↓
Frontend stores:
  - localStorage.setItem("role", received_role)
  - Redirects to Dashboard
```

**Frontend Route Protection:**
```javascript
// All admin routes have this wrapper:
<Route path="/admin-dashboard" element={
  <AdminProtectedRoute>
    <AdminDashboard />
  </AdminProtectedRoute>
} />

// AdminProtectedRoute.tsx checks:
function AdminProtectedRoute({ children }) {
  const role = localStorage.getItem("role");
  
  if (role !== "admin") {
    return <Navigate to="/" replace />;
  }
  
  return children;
}
```
